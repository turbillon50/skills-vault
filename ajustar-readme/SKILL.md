# Ajustar README despues de avanzar

Despues de CADA avance importante o version, actualizar README.md del repo con: que es la app, stack, estado actual (que jala / que falta), como correr local, env vars necesarias, como se entra (URL/acceso), y un CHANGELOG corto con la fecha y que cambio. Commit aparte: docs: update README. Mantenerlo VERDADERO (sin prometer lo que no esta). Es la cara del repo y la memoria humana del proyecto.
